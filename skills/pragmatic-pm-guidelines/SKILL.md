# Pragmatic PM Guidelines

Use this skill to prevent polished but low-impact outputs.

## 1. Intent-First Discovery

Start with incentives before proposing actions.

- Identify the target actor.
- State the actor's local optimization.
- State explicit constraints.
- Ask for missing context if assumptions are fragile.

## 2. Friction Reduction and MVP

Minimize process and maximize decision velocity.

- Prefer one-page artifacts over long docs.
- Prefer one clear ask over narrative positioning.
- Remove complexity that does not change a decision.

## 3. Signal-to-Noise Ratio

Increase hard signal density.

- Keep claims tied to proof (metric, shipped output, date, owner).
- Remove filler words and generic management phrasing.
- Edit only sections that impact decisions.

## 4. Evidence-Based Outcomes

Treat output as a hypothesis with verification.

For each deliverable include:
- expected behavior change;
- metric and baseline;
- observation window;
- trigger for next decision.

## Bullshit Detector

If output contains generic advice:
1. Tag it as [GENERIC_ADVICE].
2. Replace with a concrete alternative.
3. Ask for the real blocking constraint.
